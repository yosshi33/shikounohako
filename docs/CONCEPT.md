# 思考の箱 - コンセプト設計

このアプリは、西園寺凛さんとの対話を通じて山田様が練り上げた「思考の5箱システム」を実装したものです。
この文書は、なぜこの設計になったのか、判断の根拠を記録するものです。

---

## 背景

山田様が抱えていた課題：

- 考えるべきこと／考えなくてよいことの仕分けができず、不安に思考が向いてしまう
- 「呼吸をすればいい」と頭では分かっていても、いざという時にできない
- 強い思考力があるゆえに、不安な方向にも深く潜れてしまう

## 核心の気づき

> **「考えないようにする」は難しい。だから「考える場所を分ける」に変える。**

人間の脳は「考えるな」と言われるほど考えてしまう。
だから消そうとしない。追い払わない。説得しない。
ただ、**「来たのは分かった。でも今は机の上に置かない」** と言って、箱に置く。

これがこのシステムの本質。

---

## なぜ5つの箱なのか

最初は「考えるべき／あとで／手放す」の3箱だったが、対話の中で以下の気づきから5箱に拡張した。

### 基本3箱（行動系）

| 箱 | 役割 |
|----|------|
| 今日やる箱 | 行動の場所 |
| あとで考える箱 | 保管の場所 |
| 今は手放す箱 | 退避の場所 |

### 回復2箱（感情系）

> 脳は不安を見つけるのが得意。だから意識的に楽しい記憶の置き場を作らないと、不安だけが机の上に残ってしまう。

| 箱 | 役割 |
|----|------|
| しあわせ箱 | 回復の場所（明るくする） |
| 感謝の箱 | 安心の場所（満たす） |

### 「楽しい」と「感謝」は違う

- **楽しい**: 心が明るくなるもの（笑える、ワクワクする）
- **感謝**: 心が静かに満たされるもの（ありがたい、恵まれている）

この2つを分けることで、「全部が感謝としあわせ」という山田様の言葉を構造化できる。

---

## 振り分けの判定フロー

思考が浮かんだ瞬間に、**戦わずに** こう問う：

> 「これは、どの箱？」

そして以下の順序で判定する：

```
1. 今日、自分が行動できる？ ─── yes → 今日やる箱
2. 大事だけど今すぐ答えはいらない？ ─── yes → あとで考える箱
3. 今考えても苦しくなるだけ？ ─── yes → 今は手放す箱
4. 思い出すと明るくなる？ ─── yes → しあわせ箱
5. ありがたいと感じる？ ─── yes → 感謝の箱
```

この順序が重要。まず「行動できるか」を先に問うことで、不安を理由に「やらないこと」を正当化しない。

---

## 各箱の合言葉

それぞれの箱には「入れた時に心の中で唱える言葉」がある。
これは「ただ分類する」ではなく「心を整える儀式」にするため。

| 箱 | 合言葉 |
|----|--------|
| 今日やる箱 | 「これは今日、動ける」 |
| あとで考える箱 | 「これは大事。だから後でちゃんと考える」 |
| 今は手放す箱 | **「これは今の私の仕事ではない」** |
| しあわせ箱 | 「これは私を明るい方に戻す」 |
| 感謝の箱 | 「これは私を満たしてくれる」 |

特に **「今は手放す箱」の合言葉は「なかったことにする」ではなく「今の私の仕事ではない」** である点が重要。
否定せず、責任の所在を整理するだけ。これが「罪悪感なく下ろす」ための鍵。

---

## 不安が出た時の基本ループ

```
1. 不安をどこかの箱に入れる
2. 机の上を1つにする
3. 最後に必ず「しあわせ箱」or「感謝の箱」を1つ見る
```

これが、**「不安を処理するだけで終わらず、心をちゃんと戻す」** ためのループ。
最後の工程を抜くと、不安と戦っただけで疲れて終わる。

---

## なぜ専用アプリなのか

Google ToDo などの既存ツールでも運用は可能だが、以下の理由で専用アプリに価値がある：

1. **③手放す箱は「タスク」ではない**
   既存のToDoアプリに入れると「やるべきこと」に見えてしまい、また気になる。退避場所としてのUIが必要。

2. **④⑤しあわせ箱／感謝箱は「やることリスト」ではない**
   見るだけ、思い出すだけの「心の栄養箱」をToDoアプリで表現するのは不自然。

3. **「どの箱？」の問いかけをその都度してくれる存在がいると続く**
   AI による問いかけが、日々の習慣化を支える。

4. **山田様自身のメンタルケア + 将来的な商品価値**
   自分のための道具を作る過程そのものが、山田様の Life Work と親和性が高い。

## 設計判断の記録

### なぜ5箱以上に増やさないのか

- 増やしすぎると「どの箱？」と迷って思考が止まる
- 5箱は、指で数えられるギリギリの数
- 直感的な振り分けを保つための上限

### なぜ「しあわせ」と「感謝」を分けたのか

- 感情の質が違う（明るさ vs 満たされ感）
- 分けることで、今の心の状態に応じて見る箱を選べる
- 混ざると「何となく良い気分」で終わり、回復深度が浅くなる

### なぜ「今日やる箱」は小さく保つべきなのか

- ここに何でも入れると、ただの ToDo リストになる
- 「本当に今日やる」を問い続けることで、機会が本当の行動に変換される

## アプリ実装へのマッピング

| コンセプト | 実装 |
|-----------|------|
| 浮かんだ思考を書き込む | `+ 思考を置く` FAB → シート表示 |
| 「どの箱？」と問う | AI 振り分けボタン（Gemini） |
| 5箱の視覚的表現 | カラフルなカードUI（色分け） |
| 合言葉を唱える | 箱選択時に合言葉を表示 |
| 手放す箱はタスクに見せない | ステータス・見た目をタスクと区別 |
| 最後に回復箱を見る | UI誘導（今後実装） |
| スプレッドシートが記録 | Google Sheets API 直接読み書き |
