# 思考の箱 - shikounohako

> 考えなくてよい思考が浮かんだ時に、**置いておく場所**。
> 消そうとしない。追い払わない。説得しない。ただ、机から下ろす。

## これは何？

反すう思考（ぐるぐると同じ悩みが回る状態）を、頭の中で戦うのではなく「箱に入れる」ことで整理するための個人用メンタルケアWebアプリ。

**目的は「考えないようにすること」ではなく、考える場所を仕分けること。**

## 5つの箱

| 箱 | 役割 | 入れるもの |
|----|------|-----------|
| 1️⃣ 今日やる箱 | 行動の場所 | 今日、自分の行動で1歩進められること |
| 2️⃣ あとで考える箱 | 保管の場所 | 大事だけど今すぐ答えを出さなくていいこと |
| 3️⃣ 今は手放す箱 | 退避の場所 | 過去の後悔、最悪予測、自己否定 |
| 🌸 しあわせ箱 | 回復の場所 | 楽しい、嬉しい、好き、安心 |
| 🍀 感謝の箱 | 安心の場所 | ありがたい、支えられている、満たされている |

不安をどこかの箱に入れたら、最後に「しあわせ箱」か「感謝の箱」を一つ読む。これで、不安処理だけで終わらず、心をちゃんと戻せる。

## 主な機能

- **思考の入力と箱分け**: 浮かんだ思考を書き込むと、AI が「どの箱？」を提案
- **手動選択も可能**: AI を使わず、5つの箱から自分で選ぶこともできる
- **スプレッドシート同期**: Googleスプレッドシートがそのままデータ置き場に
- **PWA対応**: スマホのホーム画面に追加して、ネイティブアプリのように使える
- **オフライン入力**: 一度開いたら、圏外でも入力可能（次回オンライン時に同期）

## アーキテクチャ

```
[ブラウザ (PWA)]
  ├ Google OAuth ログイン
  ├ Google Sheets API (トークンで直接呼び出し)
  └ Gemini API (AI振り分け)
       ↓
[GitHub Pages] ← 静的ホスティングのみ
       ↓
[Googleスプレッドシート] ← データ本体
```

**サーバーレス・APIキー非保持**。ユーザーのGoogleアカウントのOAuthトークンでSheets/Gemini APIに直接アクセスする設計のため、APIキーの漏洩リスクがない。

## セットアップ

詳細は [docs/SETUP.md](docs/SETUP.md) を参照。

### 必要なもの

- Googleアカウント
- GitHubアカウント
- Node.js 20+

### 最短手順

1. Google Cloud で OAuth クライアント ID を作成
2. Google Sheets API と Generative Language API (Gemini) を有効化
3. GitHub リポジトリに Secrets を設定
4. main ブランチに push → GitHub Actions が自動デプロイ

## 開発

```bash
npm install
cp .env.example .env
# .env に VITE_GOOGLE_CLIENT_ID と VITE_GOOGLE_SHEETS_ID を書く
npm run dev
```

## データ構造（スプレッドシート）

アプリ起動時に `thoughts` シートが自動作成され、以下の列で管理されます：

| 列 | 内容 |
|----|------|
| A: timestamp | 登録日時 (ISO) |
| B: id | 一意ID |
| C: content | 思考内容 |
| D: box | 箱キー (`today` / `later` / `release` / `happy` / `gratitude`) |
| E: source | 振り分け元 (`ai` / `manual`) |
| F: ai_reason | AI判定理由 |
| G: status | `active` / `archived` |
| H: due_at | あとで考える箱の予約日 |
| I: reviewed_at | 最終確認日時 |
